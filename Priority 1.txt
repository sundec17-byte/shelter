---

## 1 — Persistent Cookie Session

Implement persistent cookie handling for all Python download requests.

Current:

```python
urllib.request.urlopen(req)
```

Required:

* persistent cookie jar
* cookies reused across requests
* ability to import/export browser cookies later

Suggested:

```python
import http.cookiejar

cookie_jar = http.cookiejar.MozillaCookieJar("cookies.txt")

opener = urllib.request.build_opener(
    urllib.request.HTTPCookieProcessor(cookie_jar)
)
```

All requests should use the opener object instead of raw `urlopen()`.

Persist cookies between runs.

---

## 2 — Humanised Rate Limiting

Current requests are too regular.

Add:

* random jitter
* burst cooldowns
* occasional longer pauses
* adaptive slowdown mode

Required behaviour examples:

Per-request delay:

```python
time.sleep(random.uniform(2.5, 8.0))
```

Burst cooldown:

```python
if count % random.randint(15, 40) == 0:
    time.sleep(random.uniform(90, 420))
```

Occasional random idle:

```python
if random.random() < 0.08:
    time.sleep(random.uniform(20, 90))
```

Avoid perfect timing patterns.

---

## 3 — CAPTCHA Detection

Add detection logic for:

* redirects to captcha.photoshelter.com
* HTML content instead of image/jpeg
* unexpected content-types
* unusual redirect chains

If detected:

* immediately pause queue
* stop retries
* enter cooldown mode
* preserve session state

DO NOT aggressively retry CAPTCHA-triggered requests.

---

## 4 — Queue Persistence

Implement resumable queue system.

Store:

```json
{
  "id": "...",
  "crop_downloaded": true,
  "wm_downloaded": true,
  "iptc_extracted": true,
  "composited": true,
  "last_attempt": "...",
  "fail_count": 0
}
```

Goals:

* never re-download unnecessarily
* survive interruptions
* reduce repeated requests

---

##  5 — Reduce Predictable Behaviour

Current request order is deterministic:

1. crop
2. wm
3. next image

Randomise ordering:

```python
tasks = [crop_url, wm_url]
random.shuffle(tasks)
```

Add entropy wherever reasonable.

---

##  6 — Low Concurrency

Keep concurrency extremely low.

Recommended:

* 1 request at a time
* maximum 2 simultaneous requests

Do NOT aggressively parallelise downloads.

---

##  7 — Local Cache Layer

Cache:

* downloaded renders
* extracted IPTC
* generated composites

Never re-fetch existing successful assets unless forced.

***Priority/Task 2***
some "crop" images are not complete we need to  detect the correct position on the full image to overlay
Current behaviour incorrectly assumes:

the cropped image is always aligned to the top-left of the original image
the crop always retains full original width
overlay position is always x=0, y=0

This assumption is invalid.

Some crops remove more content from the right side (and potentially left side), meaning the cropped image is offset within the original image dimensions.

A fixed overlay therefore causes:

misaligned faces
duplicated edges
visible seams
incorrect crop placement

The system must instead automatically detect the exact position of the cropped image inside the original image before compositing.

Required behaviour:

Treat the cropped image as a subsection/window extracted from the original image.
Automatically locate where the cropped image exists within the larger image.
Use the detected X/Y coordinates as the overlay origin.
Perform overlay/compositing using those coordinates instead of 0,0.

Recommended implementation:

Use template matching.

Process:

Convert both images to grayscale.
Run template matching using OpenCV.
Find the highest-confidence match position.
Use that position as the overlay offset.
Composite overlay at detected coordinates.

Example:

Incorrect:
overlayX = 0
overlayY = 0

Correct:
overlayX = detectedMatchX
overlayY = detectedMatchY

Example result:
overlayX = 148
overlayY = 0

Meaning:

the crop begins 148px from the left edge
overlay must start there

Suggested OpenCV implementation:

import cv2

base = cv2.imread("base.jpg")
overlay = cv2.imread("overlay.jpg")

base_gray = cv2.cvtColor(base, cv2.COLOR_BGR2GRAY)
overlay_gray = cv2.cvtColor(overlay, cv2.COLOR_BGR2GRAY)

result = cv2.matchTemplate(
    base_gray,
    overlay_gray,
    cv2.TM_CCOEFF_NORMED
)

_, max_val, _, max_loc = cv2.minMaxLoc(result)

x, y = max_loc

print("Best match:", x, y)
print("Confidence:", max_val)

Validation:

Require confidence threshold around 0.95+
Reject or log low-confidence matches

Important assumptions:

Both images must be rendered at identical scale
Minor JPEG compression differences are acceptable
No resizing should occur before matching

Future-proofing:
If future crops may include scaling, transforms, AI resizing, or rotation, migrate to feature matching methods such as:

ORB
SIFT
AKAZE

For the current use case, template matching is preferred because it is:

simpler
faster
highly accurate for identical-scale crops

Final required behaviour:

Detect crop position automatically
Calculate exact X/Y offset
Overlay using detected coordinates
Eliminate assumptions about left alignment or full-width crops
